# 数据校验规则文档

## 1. 校验规则概述

本文档定义系统中所有数据校验规则,确保数据完整性、准确性和一致性。

## 2. 企业备案信息校验

### 2.1 组织机构代码校验

**规则编号:** VAL-001

**校验规则:**
```
格式: 字母和数字组合
长度: 1-9位
必填: 是
唯一性: 系统内唯一
正则表达式: ^[A-Z0-9]{1,9}$
```

**错误提示:** "组织机构代码格式错误,请输入1-9位字母或数字"

### 2.2 邮政编码校验

**规则编号:** VAL-002

**校验规则:**
```
格式: 6位纯数字
必填: 是
范围: 100000-999999(有效邮编范围)
正则表达式: ^\d{6}$
```

**错误提示:** "邮政编码格式错误,请输入6位数字"

### 2.3 联系电话校验

**规则编号:** VAL-003

**校验规则:**
```
格式: 支持以下格式
  - 手机: 11位数字,1开头
  - 座机: 区号(3-4位)-号码(7-8位)
  - 400电话: 400-XXX-XXXX
必填: 是
正则表达式: ^1[3-9]\d{9}$|^0\d{2,3}-?\d{7,8}$|^400-?\d{3}-?\d{4}$
```

**错误提示:** "联系电话格式错误"

### 2.4 传真号码校验

**规则编号:** VAL-004

**校验规则:**
```
格式: 区号(3-4位)-号码(7-8位)
必填: 是
正则表达式: ^0\d{2,3}-?\d{7,8}$
```

**错误提示:** "传真号码格式错误"

### 2.5 所属地区校验

**规则编号:** VAL-005

**校验规则:**
```
类型: 下拉选择
必填: 是
可修改: 否(系统自动填充,不可修改)
```

### 2.6 企业性质校验

**规则编号:** VAL-006

**校验规则:**
```
类型: 两级下拉选择
必填: 是
一级: 企业性质分类
二级: 根据一级动态加载
```

### 2.7 行业分类校验

**规则编号:** VAL-007

**校验规则:**
```
类型: 两级下拉选择
必填: 是
一级: 行业大类
二级: 行业中类
```

## 3. 月度填报数据校验

### 3.1 就业人数校验

**规则编号:** VAL-010

**校验规则:**
```
建档期就业人数:
  - 格式: 非负整数
  - 范围: 0-999999
  - 必填: 是

调查期就业人数:
  - 格式: 非负整数
  - 范围: 0-999999
  - 必填: 是

正则表达式: ^\d+$
```

**错误提示:** "就业人数必须为非负整数"

### 3.2 条件必填校验

**规则编号:** VAL-011

**业务逻辑:**
```
IF 调查期就业人数 < 建档期就业人数 THEN
  就业人数减少类型 = 必填
  主要原因及说明 = 必填
ELSE
  就业人数减少类型 = 可选
  主要原因及说明 = 可选
END IF
```

**错误提示:** 
- "就业人数减少时,必须选择减少类型"
- "就业人数减少时,必须填写主要原因及说明"

### 3.3 减少类型校验

**规则编号:** VAL-012

**校验规则:**
```
类型: 下拉选择
必填: 条件必填(VAL-011)
可选项: 从字典动态加载
```

**减少原因字典:**
1. 产业结构调整
2. 重大技术改革
3. 关闭破产
4. 经济性裁员
5. 订单不足
6. 用工成本上升
7. 自然减员
8. 季节性用工
9. 自行离职
10. 其他

### 3.4 主要原因及说明校验

**规则编号:** VAL-013

**校验规则:**
```
类型: 文本输入
必填: 条件必填(VAL-011)
长度: 10-500字
正则表达式: ^[\u4e00-\u9fa5a-zA-Z0-9,，。.!！?？\s]{10,500}$
```

**错误提示:** "请输入10-500字的原因说明"

## 4. 密码校验规则

### 4.1 密码强度校验

**规则编号:** VAL-020

**校验规则:**
```
长度: 8-20位
必须包含:
  - 至少1个大写字母
  - 至少1个小写字母
  - 至少1个数字
不能包含:
  - 用户名
  - 连续字符(如123456, abcdef)
  - 重复字符(如111111, aaaaaa)
历史校验:
  - 不能与最近3次密码相同
正则表达式: ^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d@$!%*?&]{8,20}$
```

**错误提示:**
- "密码长度必须为8-20位"
- "密码必须包含大写字母、小写字母和数字"
- "密码过于简单,请勿使用连续或重复字符"
- "新密码不能与最近3次密码相同"

## 5. 时限校验规则

### 5.1 上报时限校验

**规则编号:** VAL-030

**校验规则:**
```
调查期开始时间 < 调查期结束时间
调查期结束时间 > 当前时间
时限设置必须由省级用户操作
```

**错误提示:** "调查期时间范围设置错误"

### 5.2 填报时间校验

**规则编号:** VAL-031

**校验规则:**
```
当前时间必须在调查期规定的时间范围内
逾期不可填报(除非省级用户开放补报)
```

**错误提示:** "当前不在填报时间范围内,请联系省级管理部门"

## 6. 数据一致性校验

### 6.1 审核状态一致性

**规则编号:** VAL-040

**校验规则:**
```
企业提交 → 市级待审核
市级通过 → 省级待复核
省级通过 → 已上报部委
任何环节退回 → 企业待修改
```

### 6.2 数据汇总一致性

**规则编号:** VAL-041

**校验规则:**
```
省级汇总数据 = SUM(各市级审核通过数据)
部委上报数据 = SUM(各省级复核通过数据)
```

## 7. 校验实现建议

### 7.1 前端校验
- 实时校验,输入时即时反馈
- 表单提交前全面校验
- 友好的错误提示

### 7.2 后端校验
- 所有前端校验规则必须重复
- 数据库层面添加约束
- 事务处理保证数据一致性

### 7.3 校验优先级
```
P0(必须): 格式校验、必填校验
P1(重要): 业务逻辑校验、一致性校验
P2(建议): 数据合理性提示
```
