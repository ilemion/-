# 系统配置管理方案

## 1. 配置管理目标

通过配置化设计,提高系统灵活性,降低政策性变更对系统的影响,实现快速响应政策调整和业务需求变化。

## 2. 配置架构设计

### 2.1 配置分层架构

```
┌─────────────────────────────────────┐
│        应用层配置(用户可操作)         │
│  - 上报时限配置                      │
│  - 通知模板配置                      │
│  - 报表格式配置                      │
├─────────────────────────────────────┤
│        业务层配置(管理员可操作)       │
│  - 数据字典配置                      │
│  - 表单字段配置                      │
│  - 校验规则配置                      │
├─────────────────────────────────────┤
│        系统层配置(运维可操作)         │
│  - 系统参数配置                      │
│  - 接口配置                          │
│  - 安全配置                          │
└─────────────────────────────────────┘
```

### 2.2 配置存储方案

**数据库配置表:**
- 动态配置(频繁变更)
- 业务配置(字典、规则)
- 支持热更新

**配置文件:**
- 静态配置(不常变更)
- 系统参数
- 支持环境变量

**配置中心(可选):**
- Apollo/Nacos
- 集中管理
- 版本控制
- 灰度发布

### 5.3 时限校验逻辑

**后端校验:**
```java
public boolean isWithinPeriod(String periodCode, Long enterpriseId) {
    SurveyPeriod period = periodMapper.selectByCode(periodCode);
    
    // 检查是否在正常填报期内
    LocalDate today = LocalDate.now();
    if (!today.isBefore(period.getStartDate()) && !today.isAfter(period.getEndDate())) {
        return true;
    }
    
    // 检查是否允许补报
    if (period.getAllowLateSubmit() && 
        !today.isAfter(period.getLateSubmitEndDate())) {
        return true;
    }
    
    return false;
}
```

### 6.3 动态校验引擎

```java
@Component
public class DynamicValidationEngine {
    
    @Autowired
    private ValidationRuleMapper ruleMapper;
    
    public ValidationResult validate(String formCode, Map<String, Object> data) {
        List<ValidationRule> rules = ruleMapper.selectByFormCode(formCode);
        
        for (ValidationRule rule : rules) {
            if (!rule.getIsActive()) {
                continue;
            }
            
            Object value = data.get(rule.getFieldCode());
            
            boolean isValid = evaluateRule(rule, value, data);
            
            if (!isValid) {
                return ValidationResult.failed(rule.getErrorMessage());
            }
        }
        
        return ValidationResult.passed();
    }
    
    private boolean evaluateRule(ValidationRule rule, Object value, Map<String, Object> data) {
        switch (rule.getRuleType()) {
            case "required":
                return value != null && !String.valueOf(value).isEmpty();
                
            case "format":
                return Pattern.matches(rule.getRuleExpression(), String.valueOf(value));
                
            case "range":
                return evaluateRangeRule(rule.getRuleExpression(), value);
                
            case "conditional":
                return evaluateConditionalRule(rule.getRuleExpression(), data);
                
            default:
                return true;
        }
    }
}
```

```
评估影响范围
    ↓
修改配置
    ↓
测试验证
    ↓
发布配置
    ↓
监控效果
```

### 8.2 变更审计

**记录内容:**
- 变更时间
- 变更人
- 变更内容
- 变更原因
- 影响范围

**审计查询:**
```sql
SELECT * FROM config_change_log 
WHERE dict_code = 'reduce_reason' 
ORDER BY change_time DESC;
```

2. 导入配置数据
3. 验证配置
4. 刷新缓存
5. 通知用户
