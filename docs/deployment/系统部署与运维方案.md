## 2. 部署流程

### 2.1 环境准备

**操作系统:**
- CentOS 7.9 或 Ubuntu 20.04
- 最小化安装
- 安全加固

**基础软件:**
- JDK 11+
- MySQL 8.0
- Redis 6.2
- Nginx 1.20

### 2.2 部署步骤

**步骤一: 数据库部署**
```bash
# 1. 安装MySQL
sudo yum install mysql-server

# 2. 配置MySQL
sudo vi /etc/my.cnf
[mysqld]
max_connections=500
innodb_buffer_pool_size=16G

# 3. 启动MySQL
sudo systemctl start mysqld

# 4. 初始化数据库
mysql -u root -p < schema.sql
mysql -u root -p < data.sql

# 5. 配置主从复制
# 主库
CHANGE MASTER TO MASTER_HOST='slave_ip', ...
# 从库
START SLAVE;
```

**步骤二: Redis部署**
```bash
# 1. 安装Redis
sudo yum install redis

# 2. 配置Redis
sudo vi /etc/redis.conf
bind 127.0.0.1
maxmemory 8gb

# 3. 启动Redis
sudo systemctl start redis
```

**步骤三: 应用部署**
```bash
# 1. 上传应用包
scp app.jar user@app_server:/opt/app/

# 2. 配置应用
vi /opt/app/application.yml

# 3. 启动应用
nohup java -jar app.jar > app.log 2>&1 &

# 4. 配置systemd服务
sudo vi /etc/systemd/system/app.service
[Unit]
Description=Yunnan Employment System
[Service]
ExecStart=/usr/bin/java -jar /opt/app/app.jar
[Install]
WantedBy=multi-user.target

# 5. 启动服务
sudo systemctl enable app
sudo systemctl start app
```

**步骤四: Nginx部署**
```bash
# 1. 安装Nginx
sudo yum install nginx

# 2. 配置Nginx
sudo vi /etc/nginx/conf.d/app.conf
upstream backend {
    server app1:8080;
    server app2:8080;
}
server {
    listen 80;
    location / {
        proxy_pass http://backend;
    }
}

# 3. 启动Nginx
sudo systemctl start nginx
```

### 3.3 告警配置

**告警级别:**
| 级别 | 定义 | 通知方式 | 响应时间 |
|------|------|---------|---------|
| P0(致命) | 系统不可用 | 电话+短信 | 15分钟 |
| P1(严重) | 核心功能异常 | 短信+邮件 | 30分钟 |
| P2(一般) | 性能下降 | 邮件+钉钉 | 2小时 |
| P3(提示) | 预警信息 | 邮件 | 8小时 |

**告警规则示例:**
```yaml
groups:
- name: system-alerts
  rules:
  - alert: HighCPUUsage
    expr: node_cpu_usage > 80
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "CPU使用率过高"
      description: "{{ $labels.instance }} CPU使用率 {{ $value }}%"
  
  - alert: HighMemoryUsage
    expr: node_memory_usage > 85
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "内存使用率过高"
```

### 4.2 迁移流程

```
数据评估
    ↓
迁移方案制定
    ↓
迁移脚本开发
    ↓
测试环境验证
    ↓
生产环境迁移
    ↓
数据验证
    ↓
迁移完成
```

### 4.3 迁移工具

**数据导出:**
```bash
# MySQL导出
mysqldump -u root -p database_name > backup.sql

# 指定表导出
mysqldump -u root -p database_name table1 table2 > tables.sql

# 带压缩导出
mysqldump -u root -p database_name | gzip > backup.sql.gz
```

**数据导入:**
```bash
# MySQL导入
mysql -u root -p database_name < backup.sql

# 带压缩导入
gunzip < backup.sql.gz | mysql -u root -p database_name
```

**Excel导入:**
```java
// 使用EasyExcel导入
EasyExcel.read(file.getInputStream(), EnterpriseData.class, 
    new EnterpriseDataListener(enterpriseMapper)).sheet().doRead();
```

### 4.4 迁移验证

**验证内容:**
- 数据完整性验证
- 数据准确性验证
- 数据一致性验证

**验证SQL:**
```sql
-- 验证数据量
SELECT COUNT(*) FROM enterprise;

-- 验证关键字段
SELECT org_code, COUNT(*) FROM enterprise GROUP BY org_code HAVING COUNT(*) > 1;

-- 验证关联数据
SELECT e.id FROM enterprise e LEFT JOIN report r ON e.id = r.enterprise_id WHERE r.id IS NULL;
```

**应对措施:**

- 准备回滚方案

## 5. 灾备方案

### 5.1 灾备策略

**备份策略:**
| 备份类型 | 频率 | 保留时间 | 存储位置 |
|---------|------|---------|---------|
| 全量备份 | 每周 | 6个月 | 本地+异地 |
| 增量备份 | 每天 | 30天 | 本地+异地 |
| 日志备份 | 每小时 | 3年 | 本地+异地 |

### 5.2 容灾架构

```
主中心(昆明)                  灾备中心(曲靖)
┌─────────────┐              ┌─────────────┐
│ 应用服务器   │   数据同步    │ 应用服务器   │
│ 数据库主库   │←───────────→│ 数据库从库   │
│ Redis       │              │ Redis       │
└─────────────┘              └─────────────┘
```

### 5.3 容灾切换

**切换条件:**
- 主中心完全不可用
- 预计恢复时间>4小时

**切换流程:**
```
1. 确认主中心不可用
2. 启动灾备中心应用
3. 提升从库为主库
4. 切换DNS/负载均衡
5. 验证系统运行
6. 通知用户
```

**切换时间目标:**
- RTO(恢复时间目标): ≤4小时
- RPO(恢复点目标): ≤1小时

### 5.4 灾备演练

**演练频率:** 每半年一次

**演练内容:**
- 数据恢复演练
- 容灾切换演练
- 备份验证

**演练报告:**
- 演练过程记录
- 问题总结
- 改进措施
