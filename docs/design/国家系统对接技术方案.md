# 国家系统对接技术方案

## 1. 对接概述

### 1.1 对接目标
实现云南省就业失业数据与国家失业监测系统的自动数据交换,确保数据及时、准确、安全地上报到国家系统。

### 1.2 对接范围
- 企业备案数据上报
- 月度就业失业数据上报
- 省级汇总数据上报
- 异常数据说明上报

### 1.3 对接原则
- 数据安全第一
- 接口标准化
- 自动化程度高
- 异常处理完善
- 可追溯可审计

## 2. 国家接口规范分析

### 2.1 接口类型

| 接口类型 | 通信方式 | 数据格式 | 频率 |
|---------|---------|---------|------|
| 数据上报接口 | HTTP/HTTPS POST | JSON/XML | 每月一次 |
| 数据查询接口 | HTTP/HTTPS GET | JSON | 按需 |
| 状态查询接口 | HTTP/HTTPS GET | JSON | 按需 |
| 认证接口 | HTTP/HTTPS POST | JSON | 每次会话 |

### 2.2 数据交换标准

**数据格式:** JSON

**字符编码:** UTF-8

**时间格式:** ISO 8601 (YYYY-MM-DDTHH:mm:ssZ)

**数据加密:** AES-256 + RSA签名

### 2.3 接口安全要求

**认证方式:**
- API Key + Secret
- JWT Token
- 数字签名

**传输加密:**
- HTTPS (TLS 1.2+)
- 敏感字段额外加密

**签名验证:**
- RSA-2048签名
- SHA-256摘要
- 时间戳防重放

## 5. 签名与加密实现

### 5.1 签名算法

**签名步骤:**
1. 构造待签名字符串
2. 使用RSA私钥签名
3. Base64编码

**伪代码:**
```java
public String generateSignature(String requestBody, String privateKey) {
    // 1. 构造待签名字符串
    String stringToSign = requestBody + timestamp;
    
    // 2. RSA签名
    Signature signature = Signature.getInstance("SHA256withRSA");
    signature.initSign(privateKey);
    signature.update(stringToSign.getBytes("UTF-8"));
    
    // 3. Base64编码
    return Base64.getEncoder().encodeToString(signature.sign());
}
```

### 5.2 数据加密

**加密步骤:**
1. 生成随机AES密钥
2. 使用AES加密数据
3. 使用RSA公钥加密AES密钥
4. 发送加密数据和加密后的AES密钥

**伪代码:**
```java
public EncryptedData encryptData(String data, String publicKey) {
    // 1. 生成AES密钥
    KeyGenerator keyGen = KeyGenerator.getInstance("AES");
    keyGen.init(256);
    SecretKey aesKey = keyGen.generateKey();
    
    // 2. AES加密数据
    Cipher aesCipher = Cipher.getInstance("AES/GCM/NoPadding");
    aesCipher.init(Cipher.ENCRYPT_MODE, aesKey);
    byte[] encryptedData = aesCipher.doFinal(data.getBytes());
    
    // 3. RSA加密AES密钥
    Cipher rsaCipher = Cipher.getInstance("RSA/ECB/PKCS1Padding");
    rsaCipher.init(Cipher.ENCRYPT_MODE, publicKey);
    byte[] encryptedKey = rsaCipher.doFinal(aesKey.getEncoded());
    
    return new EncryptedData(encryptedData, encryptedKey);
}
```

## 6. 异常处理机制

### 6.1 异常类型

| 异常类型 | 原因 | 处理方式 |
|---------|------|---------|
| 网络异常 | 网络不通 | 重试3次,间隔1分钟 |
| 认证失败 | Token过期 | 重新获取Token后重试 |
| 签名错误 | 签名不匹配 | 检查签名算法,人工处理 |
| 数据校验失败 | 数据格式错误 | 检查数据,修正后重新上报 |
| 系统异常 | 国家系统异常 | 等待恢复后重试 |

### 6.2 重试机制

**重试策略:**
```
第一次失败 → 等待1分钟 → 重试
第二次失败 → 等待5分钟 → 重试
第三次失败 → 等待30分钟 → 重试
第四次失败 → 发送告警 → 人工处理
```

**重试代码示例:**
```java
public Response sendDataWithRetry(DataRequest request, int maxRetries) {
    int retryCount = 0;
    long waitTime = 60000; // 1分钟
    
    while (retryCount < maxRetries) {
        try {
            Response response = sendData(request);
            if (response.isSuccess()) {
                return response;
            }
        } catch (Exception e) {
            log.warn("上报失败,第{}次重试", retryCount + 1, e);
        }
        
        retryCount++;
        if (retryCount < maxRetries) {
            Thread.sleep(waitTime);
            waitTime *= 5; // 指数退避
        }
    }
    
    throw new MaxRetriesExceededException("超过最大重试次数");
}
```

### 6.3 告警通知

**告警触发条件:**
- 连续3次上报失败
- Token获取失败
- 数据校验失败

**告警方式:**
- 短信通知
- 邮件通知
- 系统内告警

**告警内容:**
- 失败时间
- 失败原因
- 影响范围
- 建议处理方式
