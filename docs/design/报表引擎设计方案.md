# 报表引擎设计方案

## 1. 报表引擎目标

提供灵活、高效的报表生成能力,支持多维数据分析、多种报表格式导出,满足不同用户的报表需求。

## 2. 报表类型设计

### 2.1 报表分类

| 报表类型 | 用途 | 展示方式 | 导出格式 |
|---------|------|---------|---------|
| 对比分析报表 | 两个调查期数据对比 | 折线图+表格 | Excel/PDF |
| 趋势分析报表 | 多个调查期趋势 | 折线图+表格 | Excel/PDF |
| 取样分析报表 | 地区分布分析 | 饼图+表格 | Excel/PDF |
| 汇总统计报表 | 全省/市汇总数据 | 表格 | Excel/PDF |
| 明细数据报表 | 企业明细数据 | 表格 | Excel |
| 预警报表 | 失业预警数据 | 表格+图表 | Excel/PDF |

### 2.2 报表模板设计

```sql
-- 报表模板表
CREATE TABLE report_template (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    template_code VARCHAR(50) NOT NULL UNIQUE COMMENT '模板编码',
    template_name VARCHAR(100) NOT NULL COMMENT '模板名称',
    template_type VARCHAR(20) COMMENT '报表类型',
    description VARCHAR(500) COMMENT '模板描述',
    config_json JSON COMMENT '模板配置',
    status TINYINT DEFAULT 1 COMMENT '状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_template_code (template_code)
) COMMENT='报表模板表';

-- 报表生成记录表
CREATE TABLE report_generation_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    template_id BIGINT NOT NULL COMMENT '模板ID',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    params JSON COMMENT '报表参数',
    status VARCHAR(20) COMMENT '状态:pending/processing/completed/failed',
    file_path VARCHAR(500) COMMENT '文件路径',
    error_msg VARCHAR(500) COMMENT '错误信息',
    start_time DATETIME COMMENT '开始时间',
    end_time DATETIME COMMENT '结束时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_status (status)
) COMMENT='报表生成记录表';
```

### 4.2 报表生成流程

```
接收报表请求
    ↓
校验参数
    ↓
加载报表模板
    ↓
生成SQL查询
    ↓
执行数据查询
    ↓
数据计算和格式化
    ↓
生成报表数据
    ↓
渲染图表/表格
    ↓
返回报表结果
```

## 5. 报表导出功能

### 5.1 Excel导出

**技术方案:** Apache POI / EasyExcel

**导出流程:**
1. 查询报表数据
2. 创建Excel工作簿
3. 写入表头
4. 写入数据行
5. 设置单元格格式
6. 输出文件流

**代码示例:**
```java
public void exportToExcel(ReportData data, HttpServletResponse response) {
    // 设置响应头
    response.setContentType("application/vnd.ms-excel");
    response.setHeader("Content-Disposition", 
        "attachment; filename=report.xlsx");
    
    // 使用EasyExcel导出
    EasyExcel.write(response.getOutputStream(), ReportRow.class)
        .sheet("报表数据")
        .doWrite(data.getRows());
}
```

### 5.2 PDF导出

**技术方案:** iText / JasperReports

**导出流程:**
1. 加载报表模板
2. 填充数据
3. 生成PDF
4. 输出文件流

### 5.3 导出性能优化

**优化措施:**
- 异步生成报表
- 分批查询数据
- 流式写入文件
- 报表缓存

**异步导出流程:**
```
用户请求导出
    ↓
创建导出任务
    ↓
返回任务ID
    ↓
后台生成报表
    ↓
更新任务状态
    ↓
通知用户下载
```

## 6. 报表缓存策略

### 6.1 缓存规则

**缓存条件:**
- 相同参数的报表
- 数据更新频率低
- 访问频率高

**缓存有效期:**
- 日报: 1小时
- 月报: 24小时
- 年报: 7天

### 6.2 缓存Key设计

```
report:{template_code}:{param1}:{param2}:{user_id}
```

### 6.3 缓存失效

**失效条件:**
- 数据更新
- 缓存过期
- 手动刷新

## 8. 附录

### 8.1 报表SQL示例

**对比分析查询:**
```sql
SELECT 
    c.city_name AS dimension_value,
    SUM(CASE WHEN period = #{period1} THEN employee_count ELSE 0 END) AS employee_count_period1,
    SUM(CASE WHEN period = #{period2} THEN employee_count ELSE 0 END) AS employee_count_period2,
    SUM(CASE WHEN period = #{period2} THEN employee_count ELSE 0 END) - 
    SUM(CASE WHEN period = #{period1} THEN employee_count ELSE 0 END) AS change_count,
    ROUND(
        (SUM(CASE WHEN period = #{period2} THEN employee_count ELSE 0 END) - 
         SUM(CASE WHEN period = #{period1} THEN employee_count ELSE 0 END)) * 100.0 / 
        NULLIF(SUM(CASE WHEN period = #{period1} THEN employee_count ELSE 0 END), 0), 
        2
    ) AS change_rate
FROM employment_data d
LEFT JOIN city c ON d.city_code = c.city_code
WHERE d.period IN (#{period1}, #{period2})
    AND d.status = 'approved'
GROUP BY c.city_code, c.city_name
ORDER BY c.city_code;
```
